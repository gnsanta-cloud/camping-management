# Google 스프레드시트 자동 동기화 (약 10분)

PC·스마트폰에서 **같은 예약·매출·상품 데이터**를 Google 시트로 공유합니다.

---

## 빠른 설정 (자동 — 권장)

1. `d:\Camping\setup-google-sheets.cmd` **더블클릭**
2. Google 로그인 **허용** (최초 1회)
3. 브라우저 주소창에 나온 URL **복사 → PowerShell 창에 붙여넣기 → Enter**
4. 완료 후 **URL + 토큰**이 클립보드에 복사됨
5. 캠핑장 앱 **설정 → Google 시트 동기화**에 붙여넣기 → **연결**

---

## 수동 설정

1. [Google Sheets](https://sheets.google.com) → **빈 스프레드시트** 생성
2. 이름 예: `캠핑장관리-동기화`

---

## 2. Apps Script 붙여넣기

1. 시트 메뉴 **확장 프로그램** → **Apps Script**
2. 기본 `Code.gs` 내용 **전부 삭제**
3. 프로젝트의 `google-apps-script/camping-sync.gs` 파일 내용 **전부 복사·붙여넣기**
4. **맨 위 `SYNC_TOKEN`** 을 임의 문자열로 변경  
   예: `camping-sync-2026-mysecret`  
   (앱에도 **같은 값** 입력)

5. **저장** (Ctrl+S)

---

## 3. 웹 앱 배포

1. Apps Script 우측 **배포** → **새 배포**
2. **유형 선택** ⚙ → **웹 앱**
3. 설정:
   - **설명**: 캠핑장 동기화
   - **실행 주체**: **나**
   - **액세스 권한**: **모든 사용자** (익명 포함)
4. **배포** → **웹 앱 URL** 복사  
   예: `https://script.google.com/macros/s/AKfycb.../exec`

> 코드 수정 후에는 **배포 → 배포 관리 → ✏️ → 새 버전**으로 다시 배포해야 반영됩니다.

---

## 4. 캠핑장 앱에 연결

### PC

1. https://gnsanta-cloud.github.io/camping-management/
2. **설정 → Google 시트 동기화**
3. **웹 앱 URL** + **동기화 토큰** (2단계에서 설정한 `SYNC_TOKEN`) 입력
4. **연결**

### 스마트폰 (sites.html PWA)

1. …/sites.html → **☁️** 버튼
2. **같은 URL + 토큰** 입력 → **연결**

---

## 5. 동작

| 동작 | 설명 |
|------|------|
| 자동 업로드 | 예약·매출 저장 후 약 1초 뒤 시트에 반영 |
| 자동 불러오기 | 20초마다 + 화면 다시 볼 때 |
| 수동 | **지금 불러오기** / **지금 업로드** |
| 예약 보기 | 시트에 **예약** 탭 자동 생성 (읽기용) |

데이터 본체는 숨김 시트 `_sync`에 JSON으로 저장됩니다.

---

## 6. 문제 해결

| 증상 | 해결 |
|------|------|
| Unauthorized | 토큰이 Apps Script `SYNC_TOKEN`과 **완전히 동일**한지 확인 |
| 업로드 실패 | 웹 앱 URL 끝이 `/exec` 인지, 배포 액세스가 **모든 사용자**인지 확인 |
| 변경 반영 안 됨 | Apps Script 수정 후 **새 버전 배포** |
| CORS/네트워크 오류 | URL 오타, 시크릿 모드에서 재시도 |

---

## 7. 보안 참고

- 동기화 토큰은 **비밀번호처럼** 관리 (localStorage에 저장됨)
- 웹 앱 URL을 아는 사람도 **토큰 없이는** 접근 불가
- 예약에 이름·연락처 포함 → 시트 **공유 범위** 주의 (본인 계정만)
